# Portfolio PDF Backend Notes

`templateEngine.process("portfolio-pdf", context)` 기준으로 `src/main/resources/templates/portfolio-pdf.html`에 넣을 템플릿 계약입니다.

## Required Context Variables

기존 변수:

| Variable | Type | Source |
| --- | --- | --- |
| `projectName` | `String` | `portfolioEntity.getProject().getTitle()` |
| `generatedAt` | `String` or formatted date | `portfolioEntity.getUpdatedAt()` |
| `overview` | `String` | `portfolioEntity.getOverview()` |
| `roles` | `String` | `portfolioEntity.getRoles()` |
| `metrics` | `String` | `portfolioEntity.getMetrics()` |
| `techStacks` | `List<String>` | `techStackJson` |
| `features` | `List<FeatureDTO>` | `featuresJson` |
| `troubleshoots` | `List<TroubleshootDTO>` | `troubleshootsJson` |
| `architectureImageUrl` | `String` | `images.architecture.imageUrl` |
| `architectureDescription` | `String` | `images.architecture.description` |
| `erdImageUrl` | `String` | `images.erd.imageUrl` |
| `erdDescription` | `String` | `images.erd.description` |
| `uiImages` | `List<ImageDTO>` | `images.ui` |

새로 추가 권장하는 변수:

| Variable | Type | Example | Note |
| --- | --- | --- | --- |
| `projectPeriod` | `String` | `2026.01 - 2026.06` | 첫 페이지 요약 정보 |
| `teamSize` | `String` | `3명` | 첫 페이지 요약 정보 |
| `primaryRole` | `String` | `Frontend / PDF Builder` | 첫 페이지 요약 정보 |
| `techStackSummary` | `String` | `React, Spring Boot, MySQL` | 첫 페이지 요약 정보 |

## Suggested `generateHtml` additions

```java
String projectPeriod = resolveProjectPeriod(portfolioEntity.getProject());
String teamSize = resolveTeamSize(portfolioEntity.getProject());
String primaryRole = resolvePrimaryRole(portfolioEntity.getRoles(), portfolioEntity.getProject());
String techStackSummary = String.join(", ", techStacks);

context.setVariable("projectPeriod", projectPeriod);
context.setVariable("teamSize", teamSize);
context.setVariable("primaryRole", primaryRole);
context.setVariable("techStackSummary", techStackSummary);
```

`generatedAt`은 `LocalDateTime` 원본보다 PDF 표시용 문자열로 포맷해서 넘기는 편이 좋습니다.

```java
context.setVariable(
        "generatedAt",
        portfolioEntity.getUpdatedAt().format(DateTimeFormatter.ofPattern("yyyy.MM.dd"))
);
```

## Null Safety

현재 코드에서 아래 접근은 `images`, `architecture`, `erd`가 null이면 NPE가 납니다.

```java
images.getArchitecture().getImageUrl()
images.getErd().getImageUrl()
```

권장 처리:

```java
PortfolioImageDTO images = objectMapper.readValue(...);
ImageDTO architecture = images != null ? images.getArchitecture() : null;
ImageDTO erd = images != null ? images.getErd() : null;

context.setVariable("architectureImageUrl", architecture != null ? architecture.getImageUrl() : null);
context.setVariable("architectureDescription", architecture != null ? architecture.getDescription() : null);
context.setVariable("erdImageUrl", erd != null ? erd.getImageUrl() : null);
context.setVariable("erdDescription", erd != null ? erd.getDescription() : null);
context.setVariable("uiImages", images != null && images.getUi() != null ? images.getUi() : List.of());
```

## Header / Footer

`portfolio-pdf.html` uses openhtmltopdf page margin boxes:

- Header left: `DevLog Portfolio`
- Header right: project title
- Footer left: `Generated By DevLog Portfolio Builder`
- Footer right: `Page {current} / {total}`

openhtmltopdf should render this through:

```css
@bottom-right {
    content: "Page " counter(page) " / " counter(pages);
}
```

If `counter(pages)` is not available in the currently used openhtmltopdf version, keep `counter(page)` only or inject total pages with a two-pass generation flow.

## Font

Korean output needs a registered Korean font. Do not rely only on CSS font-family.

Recommended:

```java
builder.useFont(() -> new FileInputStream("/fonts/NotoSansKR-Regular.ttf"), "Noto Sans KR");
builder.useFont(() -> new FileInputStream("/fonts/NotoSansKR-Bold.ttf"), "Noto Sans KR", 700, BaseRendererBuilder.FontStyle.NORMAL, true);
```
